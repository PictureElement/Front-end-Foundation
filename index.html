    "i" tag

    Description

    Represents text in an "alternate voice", mood, foreign words, technical terms or fictional character thoughts. It is a good idea to use the "class" attribute to make "i" more semantic.

    Examples

    • I hope this works, he thought.
    • We ate different types of sushi last night such as Nigiri, Sashimi and Maki.
    • There are four variations of the Fox brevity word, Fox One is one of them and indicates a simulated or actual launch of a semi-active radar-guided missile.

    "b" tag

    Description

    Represents "stylistically offset" text such as keywords and text that is bold by typographic convention(and not because is important). Also, the "b" tag can be used when you want to draw attention without increasing importance. It is a good practice to use the "class" attribute on the "b" tag in order to convey additional semantic information (i.e. <b class="lead"> for the first sentence in a paragraph). The "b" tag doesn't convey semantic information; use it only when no others fit. Do not mark titles and headings using the "b" tag.

    Examples

    • The event takes place this upcoming Saturday, and over 3,000 people have already registered.

    "em" tag

    Description

    Stress emphasis. If spoken, this stress would be emphasised pronunciation on a word.

    Examples

    • Call a doctor now!
    • I love swimming

    "strong" tag

    Description

    Strong importance.

    Examples

    • Warning! This is very dangerous.

    Updated Elements

    Example:

    <!DOCTYPE html> (1)
    <html>
      <head>
        <meta charset="UTF-8"> (2)
        <link rel="stylesheet" href="foo.css"> (3)
        <script src="foo.js"></script> (4.1)
        <script>JS code</script> (4.2)
        ...
      </head>
      ...
    </html>

    Description:

    • DOCTYPE (1): set the HTML version that the browser should use. If you not specify a version number, you're telling the browser to use the latest version.
    • Meta declaration (2): specify character encoding.
    • Link tag (3): reference external CSS files.
    • Script tag (4.1): reference external JS files.
    • Script tag (4.2): run integrated JS code.

    New Elements

    Section

    Description

    A "section" represents a generic document or application section. A "div" has no semantic meaning, but a "section" element does. A "section" is used for grouping together thematically related content. You might want to replace some of your "div" elements with "section" elements, but remember to always ask yourself, "Is all of the content related?" If yes, a "section" element is an appropriate choice. Examples of sections would be chapters, numbered sections of a thesis. A "section" element is similar to an "article" element with the main difference being that the content within a "section" element doesn't necessarily need to make sense out of the context of the page.

    Example

    HTML (Prior to HTML5)

    <div class="section">
      <h2>Blog</h2>
      <p>This is the Art Gallery blog.<p>
      ...
    </div>

    HTML5

    <section>
      <h2>Blog</h2>
      <p>This is the Art Gallery blog.<p>
      ...
    </section>

    The HTML5 implementation makes the document more semantic.

    Header

    Description

    A "header" represents a group of introductory content to a web page. This will usually contain a heading, metadata(i.e. post date) and/or navigational aids(i.e. clickable logo, table of content within a "nav" element). You can include more than one "header" element on your page but it usually appears at the top of a document or section.

    Example:

    HTML (Prior to HTML5)

    <div class="header">
      <h1>Art Gallery Blog</h1>
      <p>Covering all the Art Galleries across the globe</p>
    </div>

    HTML5

    <header>
      <h1>Art Gallery Blog</h1>
      <p>Covering all the Art Galleries across the globe</p>
    </header>

    The HTML5 implementation makes the document more semantic.

    Footer

    Description

    A "footer" represents a footer (About, Contact, Privacy Policy, Terms of Use, Trademarks, Advertise, Help Center, Subscribe, Follow) for its nearest ancestor sectioning content or sectioning root element. Like the "header", the "footer" element is not position-dependent and can be put within other HTML5 elements.

    Example

    HTML (Prior to HTML5)

    <div class="footer">
      <p>© Art Gallery 2017</p>
    </div>

    HTML5

    <footer>
      <p>© Art Gallery 2017</p>
    </footer>

    The HTML5 implementation makes the document more semantic.

    Address

    Description

    The "address" element represents the contact information(i.e. author's website or email address) for an article or web page and is not intended for marking up postal addresses. The "address" element is often used within the "footer".

    HTML5

    <footer>
      <address>
        By <a href="mailto:marios.sofokleous@yandex.com">Marios Sofokleous</a>
      </address>
      <p>Copyright Marios Sofokleous 2017<p>
    <\footer>

    Aside

    Description

    An "aside" is not directly related to the content surrounding it but it expands upon it. Used to represent content that is not the primary focus of an article or page (secondary information). When used within an "article" element, its contents should be related to that particular "article". When used outside an "article" element, the "aside" contents should be specifically related to the site. "aside"elements can be used for sidebars, pull quotes or figures.

    Example

    A page sidebar that includes filters(i.e. All, Recently Added, Recently Updated, Discontinued), about(i.e. Read more), and subscribe(i.e. Enter your email...) contents.

    HTML (Prior to HTML5)

    <div class="sidebar">
      ...
    </div>

    HTML5

    <aside>
      ...
    </aside>

    The HTML5 implementation makes the document more semantic.

    Nav

    Description

    A "nav" represents a section of a page that links to other pages or to parts within the page. This element is intended for "major navigation". A "nav" element is ideal for menus and table of contents.

    Example

    A page navigation menu(i.e. Gear, Gaming, Culture, Entertainment, Science, Video, Reviews, International Edition).

    HTML (Prior to HTML5)

    <ul class="nav">
      ...
    </ul>

    HTML5

    <nav>
      <ul>
        ...
      </ul>
    </nav>

    The HTML5 implementation makes the document more semantic.

    Article

    Description

    An "article" represents a complete or self-contained part of a document that is independently reusable or distributable. The "article" element is another type of "section" used for self-contained related content. Examples of articles would be forums posts, magazine/newspaper articles, blog entries, user submitted comments or any other independent item of content. You can nest "article" elements within one another. In this case it's implied that the nested elements are related to the outer "article" element.

    Examples

    Blog post, news story, comment on a post, a review.

    HTML (Prior to HTML5)

    <div class="article">
      ...
    </div>

    HTML5

    <article>
      ...
    </article>

    The HTML5 implementation makes the document more semantic.

    Main

    Description

    "main" represents the main content of the body of a document or application. Do not include more than one "main" element on the site. Do not include the "main" element inside of an "article", "aside", "footer", "header" or "nav" element. Any content that is repeated on multiple pages should not be placed within the "main" element.

    Example

    HTML (Prior to HTML5)

    <div class="main">
      ....
    </div>

    HTML5

    <main>
      ...
    </main>

    The HTML5 implementation makes the document more semantic.

    Figure

    Description

    A "figure" represents a unit of self-contained content and if removed from the flow of the document will not affect the document's meaning. A common use of the "figure" tag is for an image within an article.

    Example

    HTML (Prior to HTML5)

    <div class="image">
      <img src="image.png" alt="My picture">
    </div>

    HTML5

    <figure>
      <img src="image.png" alt="My picture">
    </figure>

    The HTML5 implementation makes the document more semantic.

    Figcaption

    Description

    A "figcaption" represents a caption or legend for a figure.

    Example

    HTML5

    <figure>
      <img src="image.png" alt="My picture">
      <figcaption>This is a caption for the picture. </figcaption>
    </figure>

    The HTML5 implementation makes the document more semantic.

    Time

    Description

    "time" represents either a time on a 24 hour clock, or a precise date.

    Example

    HTML5

    <time>2013-09-16</time>

    We can use the "datetime" attribute to get our desired format:
    <time datetime="2013-09-16">16/09/13</time>
    <time datetime="2013-09-16">September 16, 2013</time>
    ...

    The HTML5 implementation makes the document more semantic.

    Number

    Description

    <input type="number"> represents a precise control for setting the element's value to a string representing a number. In the browser we get a convinient control to the right side of the input that allows us to increase or decrease the value. The keyboard on mobile devices changes to have a row of number keys at the top.

    Example

    HTML5

    <form>
      Enter a number (between 1 and 5):
      <input type="number" name="number" min="1" max="5">
    </form>

    Result
    Enter a number (between 1 and 5):

    Range

    Description

    Example

    HTML5

      <input type="range" name="points" min="0" max="50">
    </form>

    New Form Elements

    Datalist

    Description

    The <datalist> element specifies a list of pre-defined options for an <input> element. Users will see a drop-down list of pre-defined options and auto-complete suggestions as they input data.

    Example:

    HTML5

    <form>
      Choose an OS:
      <input list="operating systems">
      <datalist id="operating systems">
        <option value="Windows">
        <option value="macOS">
        <option value="Linux">
        <option value="Solaris">
        <option value="FreeBSD">
      </datalist>
    </form>

    Notice: The "list" and "id" attributes need to be the same. The <input> element needs to know about the <datalist> options.

    Result
    Choose an OS:

    Placeholder

    Description

    The "placeholder" attribute specifies a hint that describes the expected value of an input field. The hint is shown inside the input field and hidden when the user starts typing.

    Example

    HTML5

    <input type="text" name="first-name" placeholder="First name">
    <input type="text" name="last-name" placeholder="Last name">
    <input type="text" name="email" placeholder="email">

    Result

    Autofocus

    Description

    The "autofocus" attribute specifies that the input field should automatically get focus(active cursor) when the page loads.

    Example

    HTML5

    <form>
      Username:<input type="text" name="username" autofocus>
      Password:<input type="password" name=:"psw">
    </form>

    Result
    Username: Password:

    Required

    Description

    The "required" attribute specifies that an input field must be filled out before submitting the form. It works with the following input types: "text", "search", "url", "tel", "email", "password", "date pickers", "number", "checkbox", "radio" and "file".

    Example

    HTML5

    <form>
      Username: <input type="text" name="username" required>
      <input type="submit">
      Password: <input type="password" name="userpsw" required>
      <input type="submit">
    </form>

    Result
    Username: Password:
