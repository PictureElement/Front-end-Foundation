Updated Tags

"i" tag

  • Description: represents text in an "alternate voice" or "mood".

  • Example:

    I hope this works, he thought.

"b" tag

  • Description: represents "stylistically offset" text.

  • Example:

    The event takes place this upcoming Saturday, and over 3,000 people have already registered.

"em" tag / "strong" tag

  • Description ("em" tag): stress emphasis.

    Description ("strong" tag): strong importance.

  • Example:

    Make sure to sign up before the day of the event, September 16, 2013.

Updated Elements

Example:

<!DOCTYPE html> (1)

<html>

  <head>

    <meta charset="UTF-8"> (2)

    <link rel="stylesheet" href="foo.css"> (3)

    <script src="foo.js"></script> (4.1)

    <script>JS code</script> (4.2)

    ...

  </head>

  ...

</html>

Description:

  • DOCTYPE (1): set the HTML version that the browser should use. If you not specify a version number, you're telling the browser to use the latest version.

  • Meta declaration (2): specify character encoding.

  • Link tag (3): reference external CSS files.

  • Script tag (4.1): reference external JS files.

  • Script tag (4.2): run integrated JS code.

New Elements

Section

  • Description: a "section" represents a generic document or application section. A "div" has no semantic meaning, but a "section" element does. A "section" is used for grouping together thematically related content. You might want to replace some of your "div" elements with "section" elements, but remember to always ask yourself, "Is all of the content related?" If yes, a "section" element is an appropriate choice.

  • Example:

    HTML (Prior to HTML5)

    <div class="section">

      <h2>Blog</h2>

      <p>This is the Art Gallery blog.<p>

      ...

    </div>

    HTML5

    <section>

      <h2>Blog</h2>

      <p>This is the Art Gallery blog.<p>

      ...

    </section>

    *HTML5 makes the document more semantic!

Header

  • Description: a "header" represents a group of introductory content (Name of the page, tagline) and/or navigational aids (clickable logo). You can include more than one header on your page. The "header" usually appears at the top of a document or section.

  • Example:

    HTML (Prior to HTML5)

    <div class="header">

      <h1>Art Gallery Blog</h1>

      <p>Covering all the Art Galleries across the globe</p>

    </div>

    HTML5

    <header>

      <h1>Art Gallery Blog</h1>

      <p>Covering all the Art Galleries across the globe</p>

    </header>

    *HTML5 makes the document more semantic!

Footer

  • Description: a "footer" represents a footer (About, Contact, Privacy Policy, Terms of Use, Trademarks, Advertise, Help Center, Subscribe, Follow) for its nearest ancestor sectioning content or sectioning root element. Like the "header", the "footer" element is not position-dependent and can be put within other HTML5 elements.

  • Example:

    HTML (Prior to HTML5)

    <div class="footer">

      <p>© Art Gallery 2017</p>

    </div>

    HTML5

    <footer>

      <p>© Art Gallery 2017</p>

    </footer>

    *HTML5 makes the document more semantic!

Aside

  • Description: "aside" is not directly related to the content surrounding it but it expands upon it. Used to represent content that is not the primary focus of an article or page (secondary information) . When used within an "article" element, its contents should be related to that particular "article". When used outside an "article" element, the "aside" contents should be specifically related to the site.

  • Example: a page sidebar that includes filters(i.e. All, Recently Added, Recently Updated, Discontinued), about(i.e. Read more), and subscribe(i.e. Enter your email...) contents.

    HTML (Prior to HTML5)

    <div class="sidebar">

      ...

    </div>

    HTML5

    <aside>

      ...

    </aside>

    *HTML5 makes the document more semantic!

Nav

  • Description: a "nav" represents a section of a page that links to other pages or to parts within the page. This element is intended for "major navigation".

  • Example: a page navigation menu(i.e. Gear, Gaming, Culture, Entertainment, Science, Video, Reviews, International Edition).

    HTML (Prior to HTML5)

    <ul class="nav">

      ...

    </ul>

    HTML5

    <nav>

      <ul>

        ...

      </ul>

    </nav>

    *HTML5 makes the document more semantic!

Article

  • Description: an "article" represents a complete or self-contained part of a document that is independently reusable or distributable. The "article" element is another type of "section" used for self-contained related content.

  • Examples: blog post, news story, comment on a post, a review.

    HTML (Prior to HTML5)

    <div class="article">

      ...

    </div>

    HTML5

    <article>

      ...

    </article>

    *HTML5 makes the document more semantic!

Main

  • Description: "main" represents the main content of the body of a document or application. Do not include more than one "main" element in a document. Do not include the "main" element inside of an "article", "aside", "footer", "header" or "nav" element.

  • Example:

    HTML (Prior to HTML5)

    <div class="main">

      ....

    </div>

    HTML5

    <main>

      ...

    </main>

    *HTML5 makes the document more semantic!

Figure

  • Description: a "figure" represents a unit of self-contained content and if removed from the flow of the document will not affect the document's meaning. A common use of the "figure" tag is for an image within an article.

  • Example:

    HTML (Prior to HTML5)

    <div class="image">

      <img src="image.png" alt="My picture">

    </div>

    HTML5

    <figure>

      <img src="image.png" alt="My picture">

    </figure>

    *HTML5 makes the document more semantic!

Figcaption

  • Description: a "figcaption" represents a caption or legend for a figure.

  • Example:

    HTML5

    <figure>

      <img src="image.png" alt="My picture">

      <figcaption>This is a caption for the picture.</figcaption>

    </figure>

    *HTML5 makes the document more semantic!

Time

  • Description: "time" represents either a time on a 24 hour clock, or a precise date.

  • Example:

    HTML5

    <time>2013-09-16</time>

    We can use the "datetime" attribute to get our desired format:

    <time datetime="2013-09-16">16/09/13</time>

    <time datetime="2013-09-16">September 16, 2013</time>

    ...

    *HTML5 makes the document more semantic!
