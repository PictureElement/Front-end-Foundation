What's New

Feb 28, 2017

Improved Element Semantics

Description

HTML5 provides better-defined semantic roles for existing elements and adds new elements with useful semantics. Therefore the use of <div> elements is kept at minimum, making the page a bit more semantic, but more importantly, easier to read.

Web Application Features

Description

Features that previously were only availbale through JS have been added directly to HTML5, such as <canvas>, <video>, and <audio>. Local storage and new input types for which the browser can expose easy user interfaces (no JS code required) have also been added. HTML5 also allows JS and browser interface to run in separate threads. Furthermore unlike its predecessor, HTML5 made it extremely easy to get the user's location. These and other features made the process of developing web applications simpler and faster.

Consistency in Handling Malformed Documents

Description

HTML supports "tag soup"(i.e. <p>This is a malformed fragment of <em>HTML.</p></em>) or the ability to write malformed code that is syntactically or structurally incorrect. Prior to HTML5, there was no standard way to correct malformed code in order to be displayed correclty in the browser. Before, new browser developers had to test malformed documents in various browsers and reverse engineer their error-handling. Fortunately, HTML5 acknowledges that badly formed HTML code already exists in large quantities and will probably continue to be used and describes exactly how "badly formed code" should be treated. In other words, the handling of badly formed code now has a place in the specification itself.
