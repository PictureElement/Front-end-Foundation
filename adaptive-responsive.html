Adaptive & Responsive Design

PictureElement
May 03, 2017

Description

Both responsive and adaptive sites change appearance based on the size of the viewport. The main difference between them, is that responsive sites, respond to the size of the viewport at any given point whereas adaptive sites adapt to the size of the viewport at specific points. Furthermore, all responsive designs are considered adaptive but not all adaptive designs are considered responsive. Adaptive designs are great for targeted content but do not necessarily cover all of the universal web. In contrast, responsive designs are flexible and universal.

Responsive/Adaptive design is not simple because there are many viewport sizes. iPhone's viewport is just one.

Relative Font Sizing

Description

It's recommended to use values that are relative(i.e. em, rem). If absolute values must be used, px are preferred.

Ems: An element's font size relative to its parent's font size

HTML

<html>
  ...
  <body>
    ...
    <h5>Google is your friend. <a href="https://www.google.com">Google it now!</a></h5>
    ...
  </body>
</html>

If the desired font size for the body is 12px and you want a font size of 15px on the heading element, and a font size of 15px for the anchor element, then you have to define the following [Remember: target(px) / context(px) = result(em)]:

CSS

/* default browser font size = 16px */
html {
  font-size: 16px;
}

/* The desire font size for the body element is 12px, which is 12/16=0.75 */
body {
  font-size: 0.75em;
}

/* The desire font size for the heading element is 15px, which is 15/12=1.25 */
h5 {
  font-size: 1.25em;
}

/* The desire font size for the anchor element is 15px, which is 15/15=1 */
h5 a {
  font-size: 1em;
}

Rems: An element's font size relative to the html element's font size

HTML

<html>
  ...
  <body>
    ...
    <h5>Google is your friend. <a href="https://www.google.com">Google it now!</a></h5>
    ...
  </body>
</html>

If the desired font size for the body is 12px and you want a font size of 15px on the heading element, and a font size of 15px for the anchor element, then you have to define the following [Remember: target(px) / html-font-size(px) = result(rem)]:

CSS

/* Make font size equal to 10px in order to make calculations easier(browser default size = 16px) */
html {
  font-size: 62.5%;
}

/* The desire font size for the body element is 12px, which is 12/10=1.2s */
body {
  font-size: 1.2rem;
}

/* The desire font size for the heading element is 15px, which is 15/10=1.5 */
h5 {
  font-size: 1.5rem;
}

/* The desire font size for the anchor element is 15px, which is 15/10=1.5 */
h5 a {
  font-size: 1.5rem;
}

Viewport

Description

The viewport is the visible area of a web page and varies with the device. Before tablets and smartphones became available, web pages had a fixed size, designed only for computer screens. Those types of pages, didn't scale down for tablet and mobile screens. Therefore, we started to use layouts that scale based on the size of the viewport. Those layouts are called fluid layouts and use relative sizes instead of fixed sizes.

Setting the viewport for a variety of devices

The following meta viewport tag should be included for pages meant for various devices. The meta viewport tag, controls the page's dimensions and scaling:

HTML

<meta name="viewport" content="width=device-width, initial-scale=1">

where:

  • width=device-width instructs the page to match the screen's width in device-independent pixels by allowing the page to reflow content to match different screen sizes.
  • initial-scale=1 instructs the page to establish a 1:1 relationship between CSS pixels and device-independent pixels regardless of device orientation.

Fluid Layouts

Description

Fluid layouts, scale based on the size of the viewport. A fluid site uses percentages instead of fixed pixel widths on large sectioning elements.

Example

HTML

<body>
  <header>
    ...
    <nav>
      ...
    </nav>
  </header>
  <main>
    <article>
      <header>
        ...
      </header>
      <section>
        ...
      </section>
      ...
    </article>
  </main>
  <aside>
    ...
  </aside>
  <footer>
    ...
  </footer>
</body>

CSS

header {
width: 100%;
}

main {
  float: left;
  width: 50%;
  margin: auto;
  overflow: hidden;
}

aside {
  float: right;
  width: 30%;
}

footer {
  width: 100%;
}

Media Queries

Description

A media query is a block of CSS properties that are applied only if a certain condition is true. Examples of such conditions are the display type, width, height, orientation and resolution. Media queries allow the application of specific styles to small screens, large screens and so on, thus providing adaptability.

Syntax

Method#1: CSS

@media (query) {
  CSS properties
}

Method #2: HTML

<link rel="stylesheet" media="(query)" href="foo.css">

The most commonly used items we can query on are:

  • min-width: rules applied for any browser width greater than the value defined in the query.
  • max-width: rules applied for any browser width less than the value defined in the query.
  • min-height: rules applied for any browser height greater than the value defined in the query.
  • max-height: rules applied for any browser height less than the value defined in the query.
  • orientation=portrait: rules applied for any browser where the height is greater than or equal to the width.
  • orientation=landscape: rules applied for any browser where the width is greater than the height.

Example

HTML: media-queries.html

<!DOCTYPE html>
<html lang="en">
  <head>
    <title>Adaptive design</title>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <meta name="description" content="CSS media queries">
    <meta name="author" content="PictureElement">
    <!-- Media query #1: max-600px.css applied for any browser width less than 600px -->
    <link rel="stylesheet" media="(max-width: 600px)" href="max-600px.css">
    <!-- Media query #2: min-800px.css applied for any browser width greater than 800px -->
    <link rel="stylesheet" media="(min-width: 800px)" href="min-800px.css">
    <!--
      Media query #3: portrait.css applied for any browser where height is greater than or equal to
      the width
    -->
    <link rel="stylesheet" media="(orientation: portrait)" href="portrait.css">
    <!--
      Media query #4: landscape.css applied for any browser where width is greater than the height
    -->
    <link rel="stylesheet" media="(orientation: landscape)" href="landscape.css">
    <style>
      @media (min-width: 600px) and (max-width: 800px) {
        #main-heading {
          color: green;
        }

        #browser::after {
          content: " 600px < w < 800px";
        }
      }
    </style>
  </head>
  <body>
    <h1 id="main-heading">Media Queries</h1>
    <h2 id="browser">Browser:</h2>
    <h2 id="orientation">Orientation:</h2>
  </body>
</html>

CSS: max-600px.css

#main-heading {
  color: red;
}

#browser::after {
  content: " 0 < w < 600px";
}

CSS: min-800px.css

#main-heading {
  color: blue;
}

#browser::after {
  content: " w > 800px";
}

CSS: portrait.css

#orientation::after {
  content: " Portrait";
}

CSS: landscape.css

#orientation::after {
  content: " Landscape";
} Try it Yourself

Break Points

Description

Break points define where the design breaks down. Break points should be based on content, never on specific contexts (i.e. devices). In other words, the content itself should determine the layout. When creating break points keep in mind that lines of text should be kept to a maximum of around 70 or 80 characters long.
