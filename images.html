Images

PictureElement
Mar 16, 2017

Image Types

  1. Content images: just as necessary to the page as all of the text(e.g. product images).
  2. Layout images: not necessary to understand the content of the page(e.g. background images).
  3. User interface images: assist with the interface of the web page, but are not required to understand the content of the page.

Content Images

Description

The <img> element represents an image in the document. This element has two required attributes: src and alt. Most browsers will display the <img> element as inline-block.

Basic Syntax

HTML

<img src="images/logo.png" alt="logo" height="50" width="50">

It is recommended to place the image files in a separate folder, not in the main site folder which includes all of our HTML and CSS.

where:

  • src: specify the URL of an image.
  • alt: specify an alternate text for an image for accessibility reasons.
  • height/width: specify the height/width of an image in pixels.

Examples

  1. Insert an image

    HTML

    <img src="youtube-logo.png" alt="YouTube">

    Result

  2. Insert an image link

    HTML

    <a href="https://youtube.com>YouTube<img src="youtube-logo.png" alt="YouTube"></a>

    Result

  3. Side-by-side images

    HTML

    <ul class="images">
      <li><img src="images/google-logo.png" alt="Google logo"></li>
      <li><img src="images/yandex-logo.png" alt="Yandex logo"></li>
      <li><img src="images/amazon-logo.jpg" alt="Amazon logo"></li>
    </ul>

    Without CSS, each one of the images displays on top of each other and has a bullet point. To make the images display side-by-side and remove the bullets, we need to jump over CSS.

    CSS

    .images {
      /* Remove bullets */
      list-style-type: none;
      /* Remove padding */
      padding: 0;
    }

    .images li {
      /* Display images inline */
      display: inline;
      /* Add a padding on the left so all images aren't just pushed right against each other. */
      padding-left: 10px;
    }

    Result

Layout Images

Description

Unlike content images, layout/background images are not added in HTML but only in CSS.

Background Properties

  1. background-color

    CSS

    body {
      background-color: #5f5f5f;
    }

    Tip: you should always set a background color in case the background image fails to load. If the background image is dark, set the background to a dark color as well.

  2. background-image

    CSS

    body {
      background-image: url(images/foo.png);
    }

    Tip: check here for free background images/patterns.

  3. background-position

    CSS

    body {
      background-position: value;
    }

    where:

    • value = keyword/x% y%/xpos ypos
    • keyword = left top, left center, left bottom, right top, right center, right bottom, center top, center center, center bottom.
    • x% y%: top left corner is 0% 0%, right bottom corner is 100% 100%.
    • xpos ypos: top left corner is 0 0, units can be any CSS units(e.g. px, em, etc.).
  4. background-repeat

    CSS

    body {
      background-position: top left;
      background-repeat: repeat/repeat-x/repeat-y/no-repeat;
    }

    where:

    • repeat: repeat background image both vertically and horizontally(default).
    • repeat-x: repeat background image just in the x-directon.
    • repeat-y: repeat background image just in the y-directon.
    • no-repeat: don't repeat at all.
  5. background-size

    CSS

    body {
      background-size: width height | cover | contain;
    }

    where:

    • width height: set the image dimensions.
    • cover: scale the background image to be as large as possible so that the background area is completely covered by the background image.
    • contain: scale the image to the largest size such that both its width and its height can fit inside the content area.

Background Shorthand Property

Background properties can be shortened into a single background property:

CSS

body {
  background: bg-color bg-image bg-position/bg-size bg-repeat;
}

Note: If one of the properties in the shorthand declaration is the bg-size property, you must use a / (slash) to separate it from the bg-position property.

Writing on Top of Background Images

Setup containers that display backgrounds/layout images and then add text tags on top of them.

One way to do this, is to create a <div> container that completely contains a background image, and put text tags inside of it:

HTML

<div class="featured-image">
  <!-- Add a text tag of your choice on top of the featured image -->
  <p>Sneakers up to 30% off</p>
</div>

CSS

.featured-image {
  /* Actual width of the <div> container */
  width: 200px;
  /* Actual height of the <div> container */
  height: 100px;
  /* Background image properties */
  background: #ffffff url(images/sneakers.jpg) 0 0/200px 100px no-repeat;
}

/* Style the <p> tag by setting various properties */
.featured-image p {
  /* Pink background color */
  background-color: #ffc0cb;
  /* Green font color */
  color: #008000;
  /* Adjust the padding */
  padding: 5px 0 5px 0;
  /* Make the text all uppercase */
  text-transform: uppercase;
}

Result

Sneakers up to 30% off

Floating Images

Description

To display an image to the side of block level tags, in other words to have an image sitting to the left/right of some text we use the float property.

Example

HTML

<ul class="cakes">
  <li>
    <img src="images/strawberry-cake.jpg" alt="strawberry cake" height="90">
    <h1>Strawberry Cake</h1>
    <p>Description: This is a dangerously delicious cake!</p>
    <p>Creator: Marios Sofokleous</p>
  </li>
  <li>
    <img src="images/unknown-cake.jpg" alt="unknown cake" height="90">
    <h1>Unknown Cake</h1>
    <p>Description: Chocolate, strawberry, cookies, maltesers ... you name it!</p>
    <p>Creator: Marios Sofokleous</p>
  </li>
</ul>

CSS

.cakes {
  /* Remove bullets */
  list-style-type: none;
}

.cakes img {
  /* Float image to the left in order to wrap text around it */
  float: left;
  margin-top: 2px;
  border-radius: 50%;
  border: 1px solid black;
}

Result

  • Strawberry Cake

    Description: This is a dangerously delicious cake!

    Creator: Marios Sofokleous

  • Unknown Cake

    Description: Chocolate, strawberry, cookies, maltesers ... you name it!

    Creator: Marios Sofokleous

Note: in reality float is a tricky property, and is covered in much greater detail here.

Image Cropping

Example #1: Crop, Resize, Don't keep ratio

HTML

<img src="images/bus.jpeg" alt="bus">

CSS

img {
  border: 1px solid red;
  height: 400px;
  width: 200px;
}

Result

Example #2: Crop, Position, Don't resize, Keep ratio

HTML

<div id="crop">
  <img src="images/bus.jpeg" alt="bus">
</div>

CSS

#crop {
  border: 1px solid red;
  height: 400px;
  width: 200px;
  overflow: hidden;
}

#crop img {
  position: relative;
  left: -2500px;
  top: -1800px;
}

Result

Example #3: Crop, Position, Resize, Keep ratio, Account for portrait-oriented images

HTML

<div id="crop">
  <img src="images/bus.jpeg" alt="bus">
</div>

CSS

#crop {
  border: 1px solid red;
  height: 400px;
  width: 200px;
  overflow: hidden;
}

#crop img {
  height: 100%;
  width: auto;
  position: relative;
  left: -170px;
}

Result

Example #4: Crop, Resize, Keep ratio, Account for landscape-oriented images

HTML

<div id="crop">
  <img src="images/bus.jpeg" alt="bus">
</div>

CSS

#crop {
  border: 1px solid red;
  height: 400px;
  width: 200px;
  overflow: hidden;
}

#crop img {
  height: auto;
  width: 100%;
}

Result

Example #5: Crop, Position, Resize, Keep ratio, Account for both portrait-oriented & landscape-oriented images

HTML

<div id="crop-1" style="background-image: url(images/bus.jpeg)">
</div>
<div id="crop-2" style="background-image: url(images/selfie-monkey.jpeg)">
</div>

CSS

#crop-1 {
  width: 300px;
  height: 300px;
  /* resize image to fill the entire bg but keep ratio */
  background-size: cover;
  /* position the subject to the center */
  background-position: 35% 0;
}

#crop-2 {
  width: 300px;
  height: 300px;
  /* resize image to fill the entire bg but keep ratio */
  background-size: cover;
  /* position the subject to the center */
  background-position: 0 10%;
}

Result

Logos

Method #1: Apply logo as background

HTML

<a href="index.html" id="logo" title="Return to the homepage">Marios Sofokleous</a>

CSS

#logo {
  width: 100px;
  height: 100px;
  display: block;
  background-size: cover;
  background-image: url(images/logomakr_6AGIbQ.png);
  /* Move away text */
  text-indent: -9999px;
}

Result

Marios Sofokleous

Method #2: Apply logo as image

HTML

<a href="index.html" id="logo" title="Return to the homepage">
  <img src="images/logomakr_6AGIbQ.png" alt="logo">
</a>

CSS

#logo img {
  width: 100px;
  height: 100px;
}

Result

Note: There has been much debate for a long time as to how to mark up logos. Should your logo be applied as an image or a background? The correct answer is that logos are content (therefore images) not styles and should not be applied as backgrounds.

Sprites

Description

CSS Sprites are means of combining multiple images into a single image file to improve performance by limiting requests to the server.

Note: The following methods, make use of the following sprite:

Method #1: As background

HTML

<a href="index.html" id="logo" title="Return to the homepage">Marios Sofokleous</a>

CSS

#logo {
  width: 100px;
  height: 100px;
  display: block;
  background-size: cover;
  background-image: url(images/sprite-1.png);
  /* Move away text */
  text-indent: -9999px;
}

#logo:hover {
  background-position: 0 -100px;
}

Result

Marios Sofokleous

Method #2: As image

HTML

<a href="index.html" id="logo" title="Return to the homepage">
  <img src="images/sprite-1.png" alt="logo">
</a>

CSS

#logo {
  width: 100px;
  height: 100px;
  display: block;
  overflow: hidden;
}

#logo img {
  width: 100px;
  height: auto;
}

#logo img:hover {
  object-position: 0 -100px;
}

Result

Notes:

  1. When an image has semantical meaning(e.g. logo, product, etc.) use the second method. Otherwise, if the image is just for decoration (e.g. background of a button) use the first method.
  2. We can have the same effect with the use of two separate image files(red logo, purple logo) and by having the actual background image change on hover. However, this is a disadvantage since it requires whenever* a hover state is triggered to load an image from the server. Therefore, there's going to be a flash in the transition and an extra HTTP request(overhead).

*With the exception of preloaded images

  • height: 252 pixels

As we can infer from the sprite dimensions, each icon is 84 x 84 pixels.

<ul id="social-icons">

    </a>
  </li>
  <li>
    <a href="https://facebook.com" id="facebook">
      <img src="images/sprite-2.png" alt="social-icons-sprite">
    </a>
  </li>
  <li>
    <a href="https://linkedin.com" id="linkedin">
      <img src="images/sprite-2.png" alt="social-icons-sprite">
    </a>
  </li>
</ul>

CSS

#social-icons {
  list-style-type: none;
}

#social-icons li {
  display: inline-block;
}

#twitter, #facebook, #linkedin {
  width: 84px;
  height: 84px;
  display: block;
  overflow: hidden;
}

#twitter img {
  object-position: -672px 0;
}

#linkedin img {
  object-position: -336px 0;
}

#facebook img:hover {
  object-position: 0 -84px;
}

#twitter img:hover {
  object-position: -672px -84px;
}

#linkedin img:hover {
  object-position: -336px -84px;
}
