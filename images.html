Images

PictureElement
Mar 16, 2017

Image Types

  1. Content images: just as necessary to the page as all of the text(e.g. product images).
  2. Layout images: not necessary to understand the content of the page(e.g. background images).
  3. User interface images: assist with the interface of the web page, but are not required to understand the content of the page.

Content Images

Description

The <img> element represents an image in the document. This element has two required attributes: src and alt. Most browsers will display the <img> element as inline-block.

Basic Syntax

HTML

<img src="images/logo.png" alt="logo" height="50" width="50">

It is recommended to place the image files in a separate folder, not in the main site folder which includes all of our HTML and CSS.

where:

  • src: specify the URL of an image.
  • alt: specify an alternate text for an image for accessibility reasons.
  • height/width: specify the height/width of an image in pixels.

Examples

  1. Insert an image
    HTML

    <img src="youtube-logo.png" alt="YouTube">

    Result
  2. Insert an image link
    HTML

    <a href="https://youtube.com>YouTube<img src="youtube-logo.png" alt="YouTube"></a>

    Result
  3. Side-by-side images
    HTML

    <ul class="images">
      <li><img src="images/google-logo.png" alt="Google logo"></li>
      <li><img src="images/yandex-logo.png" alt="Yandex logo"></li>
      <li><img src="images/amazon-logo.jpg" alt="Amazon logo"></li>
    </ul>

    Without CSS, each one of the images displays on top of each other and has a bullet point. To make the images display side-by-side and remove the bullets, we need to jump over CSS.

    CSS

    .images {
      /* Remove bullets */
      list-style-type: none;
      /* Remove padding */
      padding: 0;
    }

    .images li {
      /* Display images inline */
      display: inline;
      /* Add a padding on the left so all images aren't just pushed right against each other. */
      padding-left: 10px;
    }

    Result

Layout Images

Description

Unlike content images, layout/background images are not added in HTML but only in CSS.

Background Properties

  1. background-color
    CSS

    body {
      background-color: #5f5f5f;
    }

    Tip: you should always set a background color in case the background image fails to load. If the background image is dark, set the background to a dark color as well.

  2. background-image
    CSS

    body {
      background-image: url(images/foo.png);
    }

    Tip: check here for free background images/patterns.

  3. background-position
    CSS

    body {
      background-position: value;
    }

    where:

    • value = keyword/x% y%/xpos ypos
    • keyword = left top, left center, left bottom, right top, right center, right bottom, center top, center center, center bottom.
    • x% y%: top left corner is 0% 0%, right bottom corner is 100% 100%.
    • xpos ypos: top left corner is 0 0, units can be any CSS units(e.g. px, em, etc.).
  4. background-repeat
    CSS

    body {
      background-position: top left;
      background-repeat: repeat/repeat-x/repeat-y/no-repeat;
    }

    where:

    • repeat: repeat background image both vertically and horizontally(default).
    • repeat-x: repeat background image just in the x-directon.
    • repeat-y: repeat background image just in the y-directon.
    • no-repeat: don't repeat at all.
  5. background-size
    CSS

    body {
      background-size: width height | cover | contain;
    }

    where:

    • width height: set the image dimensions.
    • cover: scale the background image to be as large as possible so that the background area is completely covered by the background image.
    • contain: scale the image to the largest size such that both its width and its height can fit inside the content area.

Background Shorthand Property

Background properties can be shortened into a single background property:

CSS

body {
  background: bg-color bg-image bg-position/bg-size bg-repeat;
}

Note: If one of the properties in the shorthand declaration is the bg-size property, you must use a / (slash) to separate it from the bg-position property.

Writing on Top of Background Images

Setup containers that display backgrounds/layout images and then add text tags on top of them.

One way to do this, is to create a <div> container that completely contains a background image, and put text tags inside of it:

HTML

<div class="featured-image">
  <!-- Add a text tag of your choice on top of the featured image -->
  <p>Sneakers up to 30% off</p>
</div>

CSS

.featured-image {
  /* Actual width of the <div> container */
  width: 200px;
  /* Actual height of the <div> container */
  height: 100px;
  /* Background image properties */
  background: #ffffff url(images/sneakers.jpg) 0 0/200px 100px no-repeat;
}

/* Style the <p> tag by setting various properties */
.featured-image p {
  /* Pink background color */
  background-color: #ffc0cb;
  /* Green font color */
  color: #008000;
  /* Adjust the padding */
  padding: 5px 0 5px 0;
  /* Make the text all uppercase */
  text-transform: uppercase;
}

Result

Sneakers up to 30% off

Floating Images

Description

To display an image to the side of block level tags, in other words to have an image sitting to the left/right of some text we use the float property.

Example

HTML

<ul class="cakes">
  <li>
    <img src="images/strawberry-cake.jpg" alt="strawberry cake" height="90">
    <h2>Strawberry Cake</h2>
    <p>Description: This is a dangerously delicious cake!</p>
    <p>Creator: Marios Sofokleous</p>
  </li>
  <li>
    <img src="images/unknown-cake.jpg" alt="unknown cake" height="90">
    <h2>Unknown Cake</h2>
    <p>Description: Chocolate, strawberry, cookies, maltesers ... you name it!</p>
    <p>Creator: Marios Sofokleous</p>
  </li>
</ul>

CSS

.cakes {
  /* Remove bullets */
  list-style-type: none;
}

.cakes img {
  /* Float image to the left in order to wrap text around it */
  float: left;
  margin-top: 2px;
  border-radius: 50%;
  border: 1px solid black;
}

Result

  • Strawberry Cake

    Description: This is a dangerously delicious cake!

    Creator: Marios Sofokleous

  • Unknown Cake

    Description: Chocolate, strawberry, cookies, maltesers ... you name it!

    Creator: Marios Sofokleous

Note: in reality float is a tricky property, and is covered in much greater detail here.
