Forms

PictureElement
Mar 21, 2017

Description

HTML forms are main points of interaction between a user and a web site. These forms are a way for a web page to get input from the user. Usually the input data is sent to the server, but sometimes can be intercepted by the web page itself. How to setup a server to receive and process the user data is beyond the scope of this article. We can still design how the from will look in the browser.

Design

The bigger the form, the more you risk losing users. Keep it simple and stay focused. Only ask for important data you need.

A simple contact form may look like this:

Implementation

HTML

<form class="myform" action="/handler-page" method="post">
  <fieldset>
    <legend>Contact Us</legend>
    <div>
      <label for="name">Name:</label>
      <input type="text" id="name" name="user-name">
    </div>
    <div>
      <label for="mail">E-mail:</label>
      <input type="email" id="mail" name="user-mail">
    </div>
    <div>
      <label for="msg">Message:</label>
      <textarea id="msg" name="user-message"></textarea>
    </div>
    <div class="button">
       <button type="submit">SEND EMAIL</button>
    </div>
  </fieldset>
</form>

  • The <form> element defines a form for collecting user input.
  • The action attribute defines the location (URL) where the form's collected data should be sent when it is submitted. In the example above, the form data is sent to a page on the server called "/handler-page". This page contains a server-side script(e.g. PHP, Python ...) that handles the form data.
  • The method attribute defines which HTTP method(GET or POST) to send the data with.
  • The <fieldset> tag is used to group related form elements.
  • The <legend> tag defines a caption for the <fieldset> element.
  • The <label> tag defines a label for an <input> element. In other words, it describes what the user should enter for that input.
  • The for attribute in the <label> tag should be the same as the value of the id attribute in an <input> tag. We do this to associate a label with an input.
  • The <input> tag specifies an input field where the user can enter data. The <input> element can be displayed in several ways depending on the type attribute. To define the default value of an <input> element you have to use the value attribute like this:
    <input type="text" value="Name">
    <input type="text" value="Surname">
  • The type attribute defines the way the <input> element behaves. In other words it specifies the type of input. Common examples of input types are:
    • <input type="text"> - single-line text field
    • <input type="checkbox"> - checkbox
    • <input type="radio"> - radio button
    • <input type="file"> - file selection field for uploads
    • <input type="password"> - password field
    • <input type="submit"> - submit button
  • The name attribute specifies the name of an <input> element. Used to reference elements in JavaScript.
  • The <textarea> tag defines a multi-line text input control. Unlike <input>, <textarea> is not an empty element, therefore it needs an opening and closing tag. To define the default value of a <textarea> element you have to put the defaut value between the starting and ending tag like this:
    <textarea>default message</textarea>
  • The <button> tag defines a clickable button.
  • The type attribute specifies the type of button. It accepts three values: button, submit, or reset.
    • button: A click on a button button does whatever was designed for. This is type is used for building custom buttons with JavaScript.
    • submit: A click on a submit button, sends the forms data to the web page defined by the action attribute.
    • reset: A click on a reset button, resets all the form elements to their default value.

Note: The main difference between <input type="submit"> and <button type="submit"> is that the former only allows plain text as its label whereas the latter allows full HTML content.

Form Styling

CSS

.myform fieldset {
  /* Form width */
  width: 400px;
  /* Outline of the form */
  border: 1px solid #ccc;
  border-radius: 10px;
  padding: 20px;
}

/* Add space between inputs */
.myform fieldset div + div {
  margin-top: 20px;
}

.myform label {
  /* Set to inline-block in order to set a width */
  display: inline-block;
  /* Same size for all labels */
  width: 90px;
  /* Align label to the right */
  text-align: right;
}

.myform input, .myform textarea {
  /*
    By default <input> fields use a sans-serif font and
    <textarea> fields use a monospace font. Make sure that all
    text fields use the same font
  */
  font-family: Lato, Helvetica, Arial;
  /* Make sure all text fields are the same size */
  width: 300px;
  */
    With border-box, the width & height properties includes content,
    padding and border, but not the margin
  */
  box-sizing: border-box;
  /* Set the same border style as the outline of the form */
  border: 1px solid #ccc;
}

/* Select and style input field when it get focus */
.myform input:focus, .myform textarea:focus {
  background-color: #ccc;
}

.myform textarea {
  /*
    Align the top of the textarea with the top of the tallest element
    on the line(i.e. label)
  */
  vertical-align: top;
  height: 100px;
}

/* Align button with text fields */
.button {
  /* Same size as the width of the labels */
  padding-left: 90px;
  /* Represents the space between a label and the text field */
  margin-left: .5em;
}
